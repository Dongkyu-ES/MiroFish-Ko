MiroFish-Ko 로컬 실행/테스트 가이드

1. 프로젝트 루트로 이동
cd /Users/dominic/Documents/github/MiroFish-Ko

2. backend 가상환경 생성 및 의존성 설치
python3.11 -m venv backend/.venv
backend/.venv/bin/python -m pip install --upgrade pip
backend/.venv/bin/python -m pip install -r backend/requirements.txt

3. .env 설정
프로젝트 루트의 .env에 아래 값을 넣는다.

LLM_PROVIDER=codex_cli
GRAPH_BACKEND=local_sqlite
CODEX_BIN=codex
CODEX_JSON_MODEL=gpt-5.4-mini
CODEX_REASONING_MODEL=gpt-5.4
CODEX_JSON_REASONING_EFFORT=high
CODEX_REASONING_EFFORT=high
CODEX_SERVICE_TIER=fast
CODEX_SANDBOX=read-only
LOCAL_GRAPH_DB_PATH=backend/uploads/local_graphs.sqlite3

참고:
- codex CLI 로그인 상태여야 한다.
- local_sqlite 모드에서는 ZEP 없이 핵심 로컬 검증이 가능하다.

4. backend 실행
backend/.venv/bin/python backend/run.py

정상 기동 확인
curl http://127.0.0.1:5001/health

정상 응답 예시
{"service":"MiroFish Backend","status":"ok"}

5. frontend 실행(선택)
npm install
cd frontend
npm install
npm run dev

frontend 주소
http://localhost:3000

backend 주소
http://localhost:5001

6. 가장 가벼운 로컬 검증 순서
(1) ontology 생성
(2) graph build
(3) simulation create
(4) simulation prepare
(5) report generate

7. 간단한 테스트용 텍스트 파일 생성
cat > /tmp/mirofish_local_graph.txt <<'EOF'
A city university changed a disciplinary policy. Students criticized the policy online. The official account posted a clarification, and local media amplified both interpretations.
EOF

8. ontology 생성 예시
curl -X POST http://127.0.0.1:5001/api/graph/ontology/generate \
  -F "files=@/tmp/mirofish_local_graph.txt" \
  -F "simulation_requirement=Forecast the next 24 hours of online public sentiment."

응답에서 project_id를 확인한다.

9. graph build 예시
curl -X POST http://127.0.0.1:5001/api/graph/build \
  -H "Content-Type: application/json" \
  -d '{"project_id":"PUT_PROJECT_ID_HERE"}'

응답에서 task_id를 확인한다.

10. graph build 상태 확인
curl http://127.0.0.1:5001/api/graph/task/PUT_TASK_ID_HERE

status가 completed가 되면 result.graph_id를 확인한다.

11. simulation create 예시
curl -X POST http://127.0.0.1:5001/api/simulation/create \
  -H "Content-Type: application/json" \
  -d '{"project_id":"PUT_PROJECT_ID_HERE","graph_id":"PUT_GRAPH_ID_HERE","enable_reddit":true,"enable_twitter":false}'

응답에서 simulation_id를 확인한다.

12. simulation prepare 예시
curl -X POST http://127.0.0.1:5001/api/simulation/prepare \
  -H "Content-Type: application/json" \
  -d '{"simulation_id":"PUT_SIMULATION_ID_HERE","use_llm_for_profiles":false,"parallel_profile_count":2}'

응답에서 task_id를 확인한다.

13. simulation prepare 상태 확인
curl -X POST http://127.0.0.1:5001/api/simulation/prepare/status \
  -H "Content-Type: application/json" \
  -d '{"task_id":"PUT_PREPARE_TASK_ID_HERE","simulation_id":"PUT_SIMULATION_ID_HERE"}'

status가 ready 또는 completed가 되면 다음 단계로 간다.

14. report generate 예시
curl -X POST http://127.0.0.1:5001/api/report/generate \
  -H "Content-Type: application/json" \
  -d '{"simulation_id":"PUT_SIMULATION_ID_HERE"}'

응답에서 report task_id와 report_id를 확인한다.

15. report 상태 확인
curl -X POST http://127.0.0.1:5001/api/report/generate/status \
  -H "Content-Type: application/json" \
  -d '{"task_id":"PUT_REPORT_TASK_ID_HERE","simulation_id":"PUT_SIMULATION_ID_HERE"}'

완료 후 보고서 조회
curl http://127.0.0.1:5001/api/report/PUT_REPORT_ID_HERE

16. start + graph memory update 검증 예시
curl -X POST http://127.0.0.1:5001/api/simulation/start \
  -H "Content-Type: application/json" \
  -d '{"simulation_id":"PUT_SIMULATION_ID_HERE","platform":"reddit","max_rounds":1,"enable_graph_memory_update":true}'

실행 상태 조회
curl http://127.0.0.1:5001/api/simulation/PUT_SIMULATION_ID_HERE/run-status

17. graph memory update 반영 확인
시뮬레이션 시작 전/후에 아래를 호출해 edge_count 변화를 본다.

curl http://127.0.0.1:5001/api/graph/data/PUT_GRAPH_ID_HERE

18. 테스트할 때 권장 사항
- report generate는 Codex 응답 시간 때문에 느릴 수 있다.
- local graph 모드에서는 report가 1섹션 compact 모드로 동작한다.
- 포트 5001이 이미 사용 중이면 기존 프로세스를 종료하고 다시 실행한다.

19. 포트 5001 정리
lsof -i tcp:5001
kill -9 PID

20. 테스트 관련 파일 위치
- 로컬 그래프 DB: backend/uploads/local_graphs.sqlite3
- simulation 산출물: backend/uploads/simulations/
- report 산출물: backend/uploads/reports/
- codex 호출 산출물: backend/uploads/codex_tasks/
